# Phase 4 – Business Pipeline & Analytics using PySpark

## Overview

This project demonstrates the implementation of an end-to-end **Business Analytics Pipeline** using **PySpark**. The pipeline performs data ingestion, cleaning, transformation, aggregation, customer segmentation, and reporting by combining customer and sales datasets.

Each analytical task is implemented using both **Spark SQL** and the **PySpark DataFrame API**, showcasing how SQL operations translate into scalable distributed data processing with Apache Spark.

---

## Objectives

- Build an end-to-end ETL and business analytics pipeline
- Perform data cleaning and validation
- Join multiple datasets using common keys
- Generate meaningful business insights from transactional data
- Implement business problems using both Spark SQL and PySpark DataFrame API
- Create a reusable reporting pipeline

## 4. Business Analytics

### Task 1 – Daily Sales

Calculate the total sales for each day.

**Output**

- Sale Date
- Total Sales

---

### Task 2 – City-wise Revenue

Calculate the total revenue generated by each city.

**Output**

- City
- Total Revenue

---

### Task 3 – Top 5 Customers

Identify the top five customers based on total spending.

**Output**

- Customer Name
- Total Spend

---

### Task 4 – Repeat Customers

Identify customers who placed more than one order.

**Output**

- Customer ID
- Order Count

---

### Task 5 – Customer Segmentation

Categorize customers based on their total spending.

**Business Rules**

- 🥇 **Gold** → Total Spend > 10,000
- 🥈 **Silver** → Total Spend between 5,000 and 10,000
- 🥉 **Bronze** → Total Spend < 5,000

**Output**

- Customer Name
- Total Spend
- Customer Segment

---

### Task 6 – Final Reporting Table

Generate a consolidated business report containing:

- Customer Name
- City
- Total Spend
- Order Count
- Customer Segment

---

### Task 7 – Save Output

Save the final reporting table for future analysis.
